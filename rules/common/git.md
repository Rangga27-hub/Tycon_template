# Common: Git

- Small, focused commits. One logical change each.
- Conventional commit style: `feat:`, `fix:`, `docs:`, `refactor:`, `test:`, `chore:`.
- Don't commit generated artifacts, secrets, or large binaries.
- Branches: `main` stable; `feat/*`, `fix/*` for work.
- Update relevant docs in the same change as the code.
