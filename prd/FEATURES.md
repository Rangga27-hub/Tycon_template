# FEATURES

> Concrete feature modules, each tagged with a priority. This bridges PRD → tasks.
> The agent turns these into the build map in `docs/engineering/PROGRESS.md`.

**Priority tags:**
- `P0` — must exist for the product to function (MVP core)
- `P1` — important, right after MVP
- `P2` — nice to have / later

> Keep each module concrete. List sub-capabilities as bullets with their own tags
> when they differ from the module's.

---

## Example (replace with your own)

### 1. Authentication `P0`
Sign in and account management.
- Email/password login `P0`
- Register `P0`
- Password reset `P1`
- OAuth (Google) `P2`

### 2. Menu / Catalog `P0`
Core content users browse.
- List items by category `P0`
- Item detail `P0`
- Search / filter `P1`

### 3. Analytics `P1`
- Sales chart by day `P1`
- Export CSV `P2`
